# 要件定義: Prompt_Generator.md チューニング（V3残存課題解消）

## 背景

V3実行結果（7/7 Pass, 0 fallback）の3世代比較および log-analyzer 分析により、
以下の品質課題が特定された。すべてプロンプト修正のみで対応可能。

## 問題

### 課題1: 括弧書き語りかけの定型化
- **発生頻度:** 7日中5〜6日で括弧書きの語りかけが出現
- **原因:** 語りかけ例に括弧パターンが多く、明示的な上限がない
- **影響:** 文体の単調化、AI感の増加

### 課題2: フック種別の「問い」偏重
- **発生頻度:** hook_strength が 0.3 に収束する傾向（pre-run で3/7Day）
- **原因:** 予告的行動・未回収出来事・伏線的具体物・身体感覚の推奨が弱い
- **影響:** hook_strength 低迷（0.8以上が全実行で0件）

### 課題3: Day5で地の文が鉤括弧に囲まれる
- **原因:** 鉤括弧の使用範囲に関する禁止ルール未記載
- **影響:** 不自然なフォーマット

### 課題4: hook_strength 3固着
- **原因:** 課題2と同根
- **対応:** 課題2の解決で自然改善を期待

## 受け入れ条件
- [ ] 括弧書き語りかけが7日間で3回以下
- [ ] フックの種類が3種以上使い分けられている（問い型が2回以下）
- [ ] 地の文が鉤括弧で囲まれている箇所がない
- [ ] hook_strength が Day 1-6 で 0.4 以上

## 影響範囲
- `prompts/Prompt_Generator.md` のみ（コード変更なし）
