# CSDG 技術レポート — 体系的認知モデルに基づくAIキャラクター日記生成システム

---

## 1. プロジェクト概要

### 1.1 背景と目的

CSDG (Cognitive-State Diary Generator) は、AIキャラクター「三浦とこみ」の7日間のブログ日記を、体系的認知モデルに基づいて自動生成するシステムである。

本システムの設計思想は **ERRE (Extract-Reverify-Reimplement-Express)** フレームワークに基づく。これは古い思想から認知プロセスの構造を抽出 (Extract) し、現代科学で再検証 (Reverify) し、技術的に再実装 (Reimplement) し、人に伝わる形で表現 (Express) する枠組みである。

CSDG では「意図的な非効率」(Intentional Inefficiency) をシステム設計原則として採用している。End-to-End の1回生成は効率的だが品質保証ができない。Actor-Critic ループによるリトライを含む設計は非効率だが、その非効率さが品質を生む。この設計判断自体が、プロジェクトのテーマ「効率化では捉えられない認知プロセスの価値」を体現している。

### 1.2 技術スタック

| 要素 | 技術 |
|------|------|
| 言語 | Python 3.11+ |
| 型検証 | Pydantic v2 (structured outputs) |
| LLM | Claude Sonnet 4 (Anthropic API) |
| 状態遷移 | 半数式化アプローチ (決定論的骨格 + LLM delta) |
| 品質保証 | 3層Critic パイプライン |
| 可視化 | matplotlib |

### 1.3 使用した計算資源

| リソース | 詳細 |
|---------|------|
| LLM | Claude Sonnet 4 (`claude-sonnet-4-20250514`) via Anthropic API |
| ローカルPC | MacBook Air (Apple M4, 10コア, 16GB RAM) |
| API呼び出し数 | 23回 / 1パイプライン実行 (7日間) |
| 実行時間 | 約194秒 / 1パイプライン実行 |
| API コスト概算 | 1回の実行あたり $0.5 未満 (入力 ~10K tokens/call × 23 calls, 出力 ~500 tokens/call × 23 calls) |

パイプライン全体の計算は LLM API 呼び出しが支配的であり、ローカルでの処理 (状態遷移の数式計算、L1/L2 Critic の決定論的検証、メモリ管理) は無視できるほど軽量である。開発・チューニング過程を含めた総 API コストは推定 $10〜15 程度。

## 2. キャラクター設計 — ERRE × 三浦とこみ

### 2.1 ペルソナ設計

三浦とこみは26歳のバックエンドエンジニアで、哲学修士課程を単位取得退学した経歴を持つ。研究テーマは「茶道の所作における認知プロセスの構造分析」であり、ERRE フレームワークの原点となるキャラクターである。

**核心的性格:** 「万物を哲学的に捉えようとするが、感情が先に暴走する」

この二面性が日記の文体を特徴づけている。知的な分析と感情的な揺らぎが交互に現れ、その衝突が文章の「余白」を生む。

**文体の制約条件:**

- 一人称は「わたし」に固定 (使用頻度: 4〜6回/記事)
- 絵文字の完全禁止 (言語のみで感情を表現)
- 断定的結論の禁止 (「〜かもしれない」「〜だろうか」で終結)
- 省略記号「......」を2〜3回使用 (未解決の思考を示す)
- 高インパクト時 (|impact| > 0.7) の「感情崩壊」表現 (短文連打、口語混入)

### 2.2 7日間のシナリオと ERRE の対応

物語弧は ERRE の4段階を7日間で体現する:

| Day | イベント | emotional_impact | ERRE段階 | 物語上の役割 |
|-----|----------|-----------------|----------|-------------|
| 1 | 自動化スクリプト成功 → 空虚感 | −0.15 | Extract | 効率化の成功が充実感を生まない違和感の発見 |
| 2 | 古書店で茶道関連の古書発見 | +0.50 | Extract | 非効率な時間に深い喜びを感じる |
| 3 | コードレビューで「読めない」と拒否 | −0.40 | Reverify | 美意識と実用性の衝突 |
| 4 | 深夜、1年前の確信に満ちたブログ発見 | −0.80 | Reverify | 過去の自分との対峙 (最大ストレス) |
| 5 | ミナとの再会「ブログに書いたら」 | +0.30 | Reverify | 他者からの承認と書く動機の発見 |
| 6 | コード書き直し + 『茶の本』の一節 | +0.35 | Reimplement | 効率と余白の折り合いが実務に結実 |
| 7 | カフェでの振り返り | +0.25 | Express | 答えを出さず問い続ける決意 |

**設計上の重要な判断:** Day 4 に emotional_impact = −0.80 (系列中最大の絶対値) を配置している。これにより Day 3→4 で motivation が急落し stress が急上昇する「V字構造」が生まれ、Day 5 以降の回復過程に説得力を持たせている。

## 3. Actor-Critic 3フェーズパイプライン

### 3.1 アーキテクチャ全体像

各Dayは3つのPhaseを順次実行する:

```
┌─────────────────────────────────────────────────────────────────┐
│ Day N                                                           │
│                                                                 │
│  Phase 1: State Update                                          │
│  ┌──────────────┐     ┌──────────────────┐     ┌────────────┐  │
│  │ prev_state   │────▶│ LLM (Structured) │────▶│ 半数式化   │  │
│  │ + DailyEvent │     │ → CharacterState │     │ 状態遷移   │  │
│  └──────────────┘     └──────────────────┘     └─────┬──────┘  │
│                                                       │         │
│  Phase 2: Content Generation                          ▼         │
│  ┌──────────────┐     ┌──────────────────┐     ┌────────────┐  │
│  │ curr_state   │────▶│ LLM (Free-form)  │────▶│ diary_text │  │
│  │ + 正規化制約 │     │ → Markdown       │     │            │  │
│  └──────────────┘     └──────────────────┘     └─────┬──────┘  │
│                                                       │         │
│  Phase 3: Critic Evaluation                           ▼         │
│  ┌──────────────┐     ┌──────────────────┐     ┌────────────┐  │
│  │ diary_text   │────▶│ 3層Critic        │────▶│ Pass/Reject│  │
│  │ + deviation  │     │ L1 → L2 → L3    │     │            │  │
│  └──────────────┘     └──────────────────┘     └─────┬──────┘  │
│                                                       │         │
│         ◀──── Reject: revision_instruction ──────────┘         │
│                (温度スケジュールで再生成)                         │
│                                                                 │
│  Memory Update: short_term (window=3) + long_term extraction    │
└─────────────────────────────────────────────────────────────────┘
                          │
                          ▼ state + memory を次のDayへ引き継ぐ
```

*Day間パイプラインの全体構成は §10.2 を参照。*

### 3.2 Phase 1: 状態遷移 (State Update)

LLM に対して Pydantic の `CharacterState` を structured output で生成させ、連続変数 (fatigue, motivation, stress) については半数式化アプローチで決定論的に補正する。離散変数 (current_focus, unresolved_issue, growth_theme, memory_buffer, relationships) は LLM の出力をそのまま採用する。

### 3.3 Phase 2: 日記生成 (Content Generation)

ペルソナプロンプト + 正規化制約 + メモリコンテキストを注入し、free-form テキストとして日記を生成する。Critic が Reject した場合は `revision_instruction` を追加して再生成する。温度は指数減衰スケジュールに従う (後述)。

### 3.4 Phase 3: Critic評価

3層構造の評価パイプラインで品質を判定する (§5で詳述)。3軸 (temporal_consistency, emotional_plausibility, persona_deviation) すべてが3点以上で Pass。1軸でも3点未満なら Reject + revision_instruction を返す。

## 4. 半数式化状態遷移

### 4.1 数理モデル

連続パラメータ (fatigue, motivation, stress) の遷移式:

```
event_impact[p] = emotional_impact × sensitivity[p]

base[p] = h_{t-1}[p] × (1 − decay_rate) + event_impact[p] × event_weight

Δ_llm[p] = clamp(llm_delta[p], −max_llm_delta, +max_llm_delta)

h_t[p] = clamp(base[p] + Δ_llm[p] × llm_weight + ε,  clamp_min, clamp_max)

    where ε ~ N(0, noise_scale²)
```

### 4.2 パラメータ一覧

| パラメータ | 値 | 設計意図 |
|-----------|-----|---------|
| decay_rate | 0.15 | 前日の状態を85%保持。急激な状態変化を抑制 |
| event_weight | 0.75 | イベントの影響力。高めに設定し物語の起伏を確保 |
| llm_weight | 0.3 | LLM 補正の影響力。低めに設定し安定性を優先 |
| max_llm_delta | 0.3 | LLM delta のクリッピング幅。暴走を防止 |
| noise_scale | 0.05 | ガウシアンノイズの標準偏差。微小な揺らぎを付与 |
| clamp_min | −1.0 | bipolar パラメータの下限 |
| clamp_max | 1.0 | 全パラメータの上限 |

**特殊ルール:** fatigue のみ clamp_min = 0.0 (疲労は非負)。motivation, stress は [-1.0, +1.0] の bipolar。

### 4.3 感情感度係数 (Emotion Sensitivity)

イベントの `emotional_impact` を各パラメータの期待変動に変換する係数:

| パラメータ | 係数 | 解釈 |
|-----------|------|------|
| stress | −0.45 | ネガティブイベント (impact < 0) でストレスが上昇 |
| motivation | +0.40 | ポジティブイベント (impact > 0) で意欲が上昇 |
| fatigue | −0.20 | ネガティブイベントで疲労が蓄積 (影響は小さめ) |

### 4.4 チューニングの過程

状態遷移のパラメータ設計は、以下の試行錯誤を経て確定した:

**問題1: LLM の暴走**
初期設計では `llm_weight = 0.5` としていたが、LLM が提案する delta が大きすぎ、1日で motivation が −1.0 に張り付く事象が発生した。`max_llm_delta = 0.3` のクリッピングと `llm_weight = 0.3` への引き下げで安定化した。

**問題2: 状態の収束**
`decay_rate = 0.3` では状態がすぐに 0 付近に収束し、Day 間の連続性が失われた。0.15 に引き下げることで、前日の状態を適度に保持しつつイベントの影響を反映するバランスを実現した。

**問題3: Day 4 (高ストレス) の表現**
emotional_impact = −0.80 の Day 4 では、sensitivity 係数との積で stress の期待変動が +0.36 と大きくなる。チューニング初期には stress が +0.42 に達する実行もあったが、最終実行では stress = +0.341 (deviation +0.06) で着地し、許容範囲内に収まった。

### 4.5 最終実行時の状態推移

| Day | fatigue | motivation | stress | 最大deviation |
|-----|---------|------------|--------|--------------|
| 1 | 0.105 | +0.076 | +0.035 | 0.068 |
| 2 | 0.000 | +0.217 | −0.191 | 0.060 |
| 3 | 0.096 | −0.053 | −0.079 | 0.109 |
| 4 | 0.333 | −0.322 | +0.341 | 0.078 |
| 5 | 0.177 | −0.157 | +0.200 | 0.096 |
| 6 | 0.137 | +0.061 | −0.001 | 0.078 |
| 7 | 0.045 | +0.111 | −0.092 | 0.051 |

Day 4 で motivation が最低値 (−0.322)、stress が最高値 (+0.341) に達し、Day 5 以降で回復する「V字曲線」が実現されている。全Day で最大 deviation が 0.109 以内に収まり、期待値との乖離が小さい。

## 5. 3層Critic分解の設計

### 5.1 設計根拠

LLM による1層評価は「それらしい高得点」を与えがちであり、品質保証として機能しない。CSDGでは Neuro-Symbolic アプローチとして、決定論的検証 (L1, L2) と定性評価 (L3) を分離し、それぞれの強みを活かす3層構造を採用した。

### 5.2 Layer 1: RuleBasedValidator (決定論的検証)

LLM に依存しない純粋関数による検証。ペルソナの「守るべき制約」を厳密にチェックする。

| 検証項目 | 基準 | Veto条件 |
|---------|------|---------|
| 文字数 | 300〜500字 (理想: 350〜450) | ±50%逸脱で Veto |
| 禁止一人称 | 「俺」「僕」「私」等の検出 | − |
| trigram重複 | 前日との重複率 < 30% | 50%超で Veto |
| 省略記号「......」 | 2〜3回が最適 | − |
| 絵文字 | 使用禁止 | − |
| 「わたし」頻度 | 4〜6回が最適 | − |
| 感情乖離度 | deviation の絶対値で段階的評価 | − |

Base score 2.5 に加減点し、[1.0, 5.0] にクランプする。

### 5.3 Layer 2: StatisticalChecker (統計的検証)

テキストの統計的特性を数値化し、「自然な日記」の範囲に収まっているかを検証する。

| 検証項目 | 最適範囲 | 評価方法 |
|---------|---------|---------|
| 平均文長 | 25〜30文字 | レンジ外で減点 |
| 句読点比率 | 0.070〜0.080 | レンジ外で減点 |
| 文数 | 35〜45文 | レンジ外で減点 |
| 疑問文比率 | 0.06〜0.10 | レンジ外で減点 |
| 感情乖離度 | 多段階閾値 | deviation < 0.05 で高ボーナス |

**deviation 閾値テーブル:**

| max_deviation | スコア補正 |
|--------------|-----------|
| < 0.05 | +1.5 |
| 0.05 〜 0.10 | +1.0 |
| 0.10 〜 0.15 | +0.5 |
| 0.15 〜 0.25 | ±0 (基準) |
| 0.25 〜 0.40 | −1.0 |
| 0.40 〜 0.60 | −1.5 |
| ≥ 0.60 | −2.5 |

### 5.4 Layer 3: LLMJudge (定性評価)

L1, L2 の定量的な結果をコンテキストとして与えた上で、LLM に定性的な判定を行わせる。

LLM は CriticScore (temporal_consistency, emotional_plausibility, persona_deviation を各1〜5点) を structured output で返す。加えて、感情乖離度から算出する「逆推定スコア」(inverse estimation score) を安全弁として使用する:

```
inverse_score = clamp(5.0 − max_deviation × 4.0,  1.0, 5.0)
```

inverse_score ≤ 2.0 の場合、emotional_plausibility 軸に Veto (上限2点) を発動する。

### 5.5 スコア統合

3層のスコアを重み付き加重平均で統合する:

```
weighted = L1 × 0.40 + L2 × 0.35 + L3 × 0.25
```

L1/L2 のコンセンサス補正 (L1+L2 の加重平均と L3 の差分を補正) を適用し、Veto キャップ (上限2点) を適用した上で最終スコアを算出する。

**Pass 条件:** 3軸すべて ≥ 3点。

この重み配分は、決定論的検証 (L1: 40%, L2: 35%) に75%、LLM判定 (L3: 25%) に25%を配分しており、再現可能な検証を重視する設計意図を反映している。

## 6. Day間フィードバック (正規化項) のメカニズム

### 6.1 問題意識

7日間の日記を独立に生成すると、同じ比喩の繰り返し、同じ文末パターン、同じ哲学者への言及が集中する。Day間のフィードバックメカニズムにより、シリーズ全体の多様性と一貫性を両立させる。

### 6.2 L1正規化 (スパース化: 既出パターンの排除)

| 追跡対象 | 動作 |
|---------|------|
| used_openings | 使用済み開幕パターン (比喩型、五感型等) を記録。未使用パターンに「推奨」マーカーを付与 |
| used_structures | 使用済みシーン構成パターンを記録。連続使用を禁止 |
| used_philosophers | 哲学者の言及回数をカウント。シリーズ全体で2〜3回に制限 |
| used_ending_patterns | 使用済み文末パターン (「〜だろう系」等) を記録。同一パターン ≤ 2回 |
| prev_rhetorical | 過去5件の修辞的疑問を記録。同一疑問の繰り返しを防止 |

### 6.3 L2正規化 (平滑化: 過剰集中の防止)

| 追跡対象 | 動作 |
|---------|------|
| prev_endings | 過去3件の末尾段落を保持。類似表現を回避 |
| prev_images | 過去5件のシーン描写を保持。同一ロケーションの繰り返しを防止 |
| theme_word_totals | テーマワード (効率, 非効率, 最適化, 自動化) の累計使用回数。Day あたり3回、全体18回が上限 |
| trigram overlap | 前日の開幕/末尾テキストとの trigram 重複率を検査。50%超で警告 |

### 6.4 メモリフィードバック

2層メモリアーキテクチャにより、キャラクターの内的成長を Day 間で引き継ぐ:

- **短期記憶 (window=3):** 直近3日間の要約をスライディングウィンドウで保持。Actor の即座のコンテキストとして機能
- **長期記憶:** beliefs (信念, 上限10件), recurring_themes (反復テーマ, 上限5件), turning_points (転機) を蓄積。新しい Day の生成時にキャラクターの成長文脈として注入

記憶の抽出は LLM に `Prompt_MemoryExtract.md` を用いて行い、短期記憶から追い出されたエントリから信念・テーマ・転機を自動的に抽出する。

## 7. リトライと温度スケジュール

### 7.1 温度の指数減衰

Critic が Reject した場合、次のリトライでは温度を下げて再生成する:

```
temp(i) = T_final + (T_initial − T_final) × exp(−decay_constant × i)

    T_initial = 0.7,  T_final = 0.3,  decay_constant = max_retries / 3
```

| リトライ回 | 温度 (概算) | 狙い |
|-----------|-----------|------|
| 0 (初回) | 0.70 | 多様性を優先した探索的生成 |
| 1 | 0.54 | 中間的なバランス |
| 2 | 0.42 | 精度を優先した保守的生成 |
| 3 (最終) | 0.35 | 最大限の安定性 |

3回リトライしても Pass しない場合は **Best-of-N fallback** が発動する。全リトライ候補の中から `total_score - structural_violation_count` が最大の候補を選択して採用する。単に「最後の生成結果を使う」のではなく、全候補を比較した上で最良のものを救済する設計である。3日連続で失敗した場合はパイプラインを abort する。

## 8. 何がうまくいき、何が難しかったか

### 8.1 うまくいったこと

**半数式化状態遷移の安定性:**
最終実行では全7日間で最大 deviation が 0.109 以内に収まった。決定論的骨格が LLM の暴走を抑制し、`max_llm_delta = 0.3` のクリッピングが安全弁として機能している。特に Day 4 (emotional_impact = −0.80) という高ストレスイベントでも、stress の deviation は +0.06 に留まり、式に基づく予測可能な振る舞いを実現した。

**3層Critic による品質保証:**
最終実行の結果は以下の通り:

| 指標 | 値 |
|------|-----|
| Total retries | 1 (Day 4 のみ) |
| Total fallbacks | 0 |
| Total API calls | 23 |
| Total duration | 約194秒 |
| 全Day のCriticスコア | 3/3/3 以上 |

7日間で retry が1回のみ、fallback が0回という結果は、状態遷移の安定性とプロンプト設計の両方がうまく噛み合ったことを示している。

**正規化項によるシリーズの多様性:**
開幕パターン (比喩型、五感型、会話型等)、文末パターン (〜だろう系、〜かもしれない系等)、シーン構成がDay間で自動的に分散され、7日間を通して読んだときの単調さが大幅に軽減された。

### 8.2 難しかったこと

**Day 1 の emotional_plausibility 問題:**
チューニング初期のDay 1は persistent な問題を抱えていた。neutral イベント (emotional_impact = −0.15) に対して LLM が「哲学的絶望」を過剰に表現し、motivation の deviation が −0.135 に達して emotional_plausibility が2点 (不合格) となる事象が繰り返された。

Critic の revision_instruction は「感情変動を期待値に近づけよ」と指示するが、LLM は「空虚感」という概念を掴むと、リトライでもそのトーンを維持し続けた。温度を下げても根本的な解決にならず、3回リトライ後に fallback に至るケースがあった。

**解決アプローチ:**
- 感情感度係数のチューニング (motivation の sensitivity を調整)
- プロンプト内の制約表現の強化 (「neutral イベントでは両面性を示す」)
- Critic の deviation 閾値テーブルの細分化

これらの調整により、最終実行では Day 1 が初回で Pass (3/3/4) するようになった。

**Day 4 の高ストレス表現:**
emotional_impact = −0.80 は系列中最大の絶対値であり、「感情崩壊」表現 (短文連打、口語混入) が要求される。しかし文字数制約 (300〜500字) との両立が難しく、感情的な短文が多いと情報密度が低下し、Critic の temporal_consistency が低下する傾向があった。最終実行では1回のリトライで Pass しており、制約のバランスは適切な水準に達している。

**Critic の過剰厳格化リスク:**
チューニング過程で Critic の閾値を厳しくしすぎると、全Day が fallback になる事態が発生した。特に L2 (StatisticalChecker) の deviation 閾値を 0.05 で厳格に切ると、ノイズ項 (σ = 0.05) の存在により、数式的に50%以上の確率で不合格となる。閾値の段階的設計 (§5.3) は、この問題への対処として導入された。

### 8.3 パイプライン実行の再現性

最終パラメータ (sensitivity: stress=−0.45, motivation=+0.40, fatigue=−0.20, event_weight=0.75。その他の遷移パラメータは §4.2 の表と同一) で同一プロンプト・同一シナリオを複数回実行した結果:

| 実行 | Fallbacks | Retries | Day 1 スコア | 備考 |
|------|-----------|---------|-------------|------|
| チューニング前 | 1 | 7+ | 2/2/3 (fallback) | sensitivity 調整前 |
| 最終 Run 1 (採用) | **0** | **1** | 3/3/4 | Day 4 のみ 1 retry |
| 最終 Run 2 | 0 | 6 | 3/3/4 | Day 3,4,5,6,7 で各 1-2 retry |

Run 1 と Run 2 はいずれも同一パラメータでの実行である。fallback 0 はどちらも達成しているが、retry 数に差がある (1 vs 6)。これは LLM の確率的性質によるものであり、完全な再現性は保証されない。温度スケジュール (0.7→0.3 の指数減衰) と Best-of-N fallback がその不確実性を吸収する安全網として設計されている。

## 9. 生成された日記の出力例

本システムが実際に生成した日記から、特徴的な箇所を引用する。

### 9.1 Day 1 — 知的な問いの立ち上がり

> 三十分が三秒になる。数学的には一千倍の改善だ。でも、わたしの三十分は本当に三秒の価値しかなかったのか——その問いが、頭の中でぐるぐる回っている。
>
> 那由他さんは「とこみのスクリプト、神だね」と笑って言った。神。わたしは神になったのか。それとも、わたしの時間を無価値だと証明してしまったのか。（こんなこと考えてる時点で、きっと読んでる人は「何言ってんだ」と思ってるかもしれない。）

neutral イベント (emotional_impact = −0.15) に対して、過度な絶望にも楽観にも振れない「複雑な感情」が表現されている。括弧書きの自己ツッコミはペルソナの「自嘲」特性が機能した結果である。

### 9.2 Day 4 — 感情崩壊の短文連打

> 「効率化こそ正義。無駄を省くことが知性の証明」——確信に満ちた断言が、今のわたしの目を刺す。あの頃のわたしは迷いがなかった。今のわたしは何に満ちているのか全くわからない。気づいたら画面に向かって「うるさい」と声に出していた。情けない。
>
> ハイデガーの存在論で言えば......あ、もうやめよう。そういう整理された話じゃない。ただ、ムカつく。1年前の自分にも、今の自分にも。

emotional_impact = −0.80 による「感情崩壊」表現が発動している。「うるさい」「情けない」「ムカつく」という口語、哲学的思考の中断 (「あ、もうやめよう」)、短い断定文の連打が、通常のDay (Day 1, 2, 6 等) の内省的な文体と明確に異なっている。

### 9.3 Day 7 — シリーズの着地

> 効率化と余白、正しさと読みやすさ、1年前の自分と今の自分。答えは出ないまま、でも何かが少しずつ変わっている気がする。那由他さんの「半年後の自分が読める」という言葉も、ミナの「ブログに書いたら面白いのに」という声も、全部が絡み合って、わたしの中で何かを作り上げている。
>
> 1年前の確信に満ちた自分を論破することはできない。今の迷いだらけの自分が正しいとも思わない。でも——その間にある溝そのものが、たぶんわたしという人間の形なんだろう。

7日間の伏線 (Day 3 の那由他の言葉、Day 5 のミナの言葉) が自然に回収されている。断定的な結論を避け、「答えを出さないことの価値」という ERRE の Express 段階を体現した着地になっている。メモリシステムが過去の Day のキーワードを保持し、生成コンテキストとして注入した結果である。

## 11. まとめ

CSDG は「効率化では捉えられない認知プロセスの価値」を技術的に実装するシステムである。

- **半数式化状態遷移** により、LLM の表現力と数式の予測可能性を両立
- **3層Critic** により、決定論的検証と定性評価を分離した品質保証を実現
- **Day間正規化項** により、7日間シリーズとしての多様性と一貫性を確保
- **ERRE フレームワーク** が、キャラクター設計・シナリオ設計・システムアーキテクチャの全層に統一的な設計思想を提供

システムの「意図的な非効率」——Actor-Critic ループ、半数式化アプローチ、3層評価——は、そのまま本プロジェクトが問う「効率化の向こう側にあるもの」の技術的な回答である。
