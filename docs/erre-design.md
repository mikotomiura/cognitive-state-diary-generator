# ERRE 設計文書 — CSDGにおける思想的フレームワークの具体化

> **目的:** CSDG が Extract-Reverify-Reimplement-Express (ERRE) フレームワークの
> 具体的実装であることを示し、技術設計と思想的設計の対応関係を明示する。

---

## 1. ERRE フレームワークとは

### 1.1 概要

ERRE は以下の4段階から成る思想的フレームワークである:

- **Extract（抽出）:** 古い思想から「人間の思考を深化させる構造」を仮説として取り出す
- **Reverify（再検証）:** 抽出した仮説を現代科学の手法で再検証する
- **Reimplement（再実装）:** 検証された知見をシステムとして動作可能な形に落とし込む
- **Express（表現）:** 再実装された知見を、人に伝わる形で表現する

ERRE は「意図的な非効率」(Intentional Inefficiency) の実践手法でもある。
効率化が捨象する認知プロセスの中に価値を見出し、それを意図的に保持・増幅するための枠組みである。

### 1.2 なぜ ERRE が必要か

AI 時代において、効率化は手段としての価値を急速に高めている。しかし「効率化では捉えられない認知プロセス」——すなわち、問いを立てる、揺らぎの中で思考を深める、答えのない状態に留まる——といった営みは、効率化の対極にある。

ERRE は古い思想の「再発見」ではない。**「再検証」と「再実装」**であるという点が重要である。過去の知恵をそのまま復活させるのではなく、現代の科学的手法で検証し、技術的に実装可能な形に翻訳する。これが ERRE の核心である。

## 2. CSDG における ERRE の具体化

### 2.1 キャラクター設計と ERRE の対応

三浦とこみの7日間の物語弧が ERRE の4段階を体現している:

| ERRE 段階 | 物語弧 | 該当 Day | キャラクターの内面 |
|-----------|--------|----------|-------------------|
| Extract (抽出) | 日常の中から「問いを立てることの価値」を抽出 | Day 1-2 | 効率化の成功が充実感を生まない違和感。古書店での非効率な時間に深い喜びを感じる |
| Reverify (再検証) | 衝突と揺らぎを通じて仮説を検証 | Day 3-5 | 問いの価値を否定され (Day 3)、社会全体のイデオロギーに直面し (Day 4)、他者の言葉で「表現者」としての自覚が芽生える (Day 5) |
| Reimplement (再実装) | 過去の研究との接続、実務への翻訳 | Day 6 | 大学院時代の現象学ノートと現在の違和感が接続され、暗黙知の可視化というアイデアに結実する |
| Express (表現) | 問いを仕事に接続する一歩 | Day 7 | 暗黙知の可視化提案を職場に持ち込む。劇的な解決ではなく「小さな一歩」であること自体が ERRE の本質 |

**重要:** Day 7 でとこみは「答え」を得ていない。ERRE の Express は「正解の提示」ではなく「問いを表現可能な形にすること」であり、とこみの日記全体がこのプロセスの記録である。

### 2.2 アーキテクチャと ERRE の対応

システムの技術的構造も ERRE を反映している (architecture.md §3 参照):

| ERRE 段階 | アーキテクチャ要素 | 設計意図 |
|-----------|-------------------|---------|
| Extract | CharacterState の連続変数 + DailyEvent | 感情の構造を「抽出」し、数値として扱えるようにする |
| Reverify | Critic 3層パイプライン (RuleBased/Statistical/LLMJudge) | LLM の生成を決定論的検証で「再検証」する。Neuro-symbolic アプローチ |
| Reimplement | Actor-Critic ループ + Self-Healing | 検証結果をリトライ・フィードバックとして「再実装」し、品質を向上させる |
| Express | 日記テキスト出力 + Prompt_Generator.md の文体制御 | キャラクターの内面を自然言語として「表現」する |

### 2.3 正規化項と ERRE

v6 で導入された正規化項アーキテクチャも ERRE の視点で理解できる (architecture.md §3.4 参照):

- **L1正規化 (スパース化):** used_openings, used_structures, used_philosophers, used_ending_patterns, prev_rhetorical
  → 過去の表現パターンから「使い古されたもの」を **Extract** し、排除する
- **L2正規化 (平滑化):** prev_endings, prev_images, theme_word_totals
  → 表現の過剰集中を **Reverify** し、多様性を確保する
- **Dropout的正規化:** 各パターンの使用上限による強制多様化
  → 制約の中で新しい表現を生み出すことを強制する (**Reimplement**)
- **事後評価 (Critic):** リトライの安全弁
  → 品質を保証した上で表現を出力する (**Express**)

## 3. 「意図的な非効率」の設計的意味

### 3.1 なぜ Actor-Critic であって End-to-End ではないのか

End-to-End の1回生成は「効率的」だが、キャラクターの一貫性と品質を保証できない。
Actor-Critic ループはリトライを含むため「非効率」だが、その非効率さが品質を生む。

これは CSDG のテーマである「意図的な非効率」をシステム設計レベルで体現している:

- **効率を追求するなら**: 1回生成 + 後処理フィルタ
- **品質を追求するなら**: Actor-Critic ループ + Self-Healing
- CSDG は後者を選択し、その設計判断自体がプロジェクトの思想を反映している

### 3.2 なぜ構造化された状態遷移なのか

LLM に全てを任せれば「効率的」だが、状態の一貫性が保証できない。
半数式化アプローチ (決定論的骨格 + LLM delta) は計算量が増える「非効率」な設計だが、
bounded dynamics による安定性と、LLM の表現力を両立させる。

## 4. 今後の展開可能性

ERRE フレームワークは CSDG に限定されず、以下の領域に適用可能:

- **教育システム:** 古典的教授法から有効な認知構造を Extract し、現代の学習科学で Reverify し、
  適応学習システムとして Reimplement し、学習者に適切な形で Express する
- **UI/UX 設計:** 伝統的な職人技のインタラクションパターンを Extract し、
  認知心理学実験で Reverify し、デジタルインターフェースとして Reimplement する
- **組織設計:** 徒弟制度の暗黙知伝達メカニズムを Extract し、
  知識経営論で Reverify し、ナレッジマネジメントシステムとして Reimplement する
